# m3508i

本项目是对大疆 RoboMaster S1 使用的 **M3508I 无刷电机** 通信协议的逆向分析成果。通过捕获控制板与电机之间的 RS485 数据包，解析了控制指令的帧格式，并提供了可生成兼容数据包的 C 语言实现。项目包含协议生成代码 (`m3508i.c`/`m3508i.h`)、RS485 转接板硬件设计 (`rs485-converter/`) 以及 STM32 示例工程 (`m3508i-test/`)，帮助开发者轻松驱动 M3508I 电机。

## 电机基本信息

- **型号**：M3508I
- **通信接口**：RS485（半双工）
- **波特率**：921600
- **供电电压**：3S 锂电池（约 12V）
- **ID 范围**：0~3（最多 4 个电机共用总线）
- **转速范围**：-1000 rpm ~ +1000 rpm
- **使能控制**：`enable` 字段控制电机是否输出力矩。`enable = false` 时，电机处于禁用状态，可自由转动无阻力。
- **自动保护**：若超过 1 秒未收到有效指令，电机自动进入禁用状态，防止控制失效时机器失控。

## 协议格式

控制板向电机发送的每一帧数据长度为 **22 字节**，结构如下：

| 偏移 | 长度 | 内容         | 说明                                                             |
|------|------|--------------|------------------------------------------------------------------|
| 0    | 7    | 帧头         | 固定值 `0x55 0x16 0x00 0x9D 0xA0 0x00 0x00`                      |
| 7    | 1    | 应答电机 ID  | 指定本次通信中哪个电机需要回复                                   |
| 8    | 8    | 电机速度数据 | 4 个电机各占 2 字节，采用小端（Little-Endian）字节序，格式见下文 |
| 16   | 4    | 填充         | 固定为 `0xFF 0xFF 0xFF 0xFF`                                     |
| 20   | 2    | CRC16        | 对前 20 字节计算的校验和，16 位值采用小端字节序，算法见后        |

> **注意**：帧是广播形式发送，总线上所有电机都会立即更新自己的速度设定，但只有 `应答电机 ID` 指定的电机会回复一个应答数据包（本项目未涉及应答包格式）。

### 速度数据编码

每个电机对应 2 字节的速度数据，其 16 位结构定义如下（采用小端字节序存放）：

| 位域   | 名称     | 说明                                                    |
|--------|----------|---------------------------------------------------------|
| bit15  | 保留     | **必须为 0**                                            |
| bit14  | 禁用标志 | 1：电机禁用（无输出力矩）；0：电机使能                  |
| bit13–0| 转速编码 | 14 位有符号整数（二进制补码），表示目标转速的线性映射值 |

数据由 `m3508i_cmd` 结构体中的 `enable` 和 `speed_rpm` 字段生成：

```c
struct m3508i_cmd {
    bool enable;       /* true: 使能电机；false: 禁用电机 */
    float speed_rpm;   /* 目标转速，范围 -1000.0 ~ 1000.0 rpm */
};
```

**编码规则：**

- **禁用电机**（`enable = false`）：  
  将 bit14 置为 1，低 14 位可任意（实际代码中保留转速计算值，但电机仅识别 bit14）。  
  此时电机不输出力矩，可自由转动。

- **使能电机**（`enable = true`）：  
  bit14 = 0，低 14 位为一个 14 位有符号整数，由转速线性映射得到：
  1. 将转速范围 `-1000 ~ 1000 rpm` 线性映射到 `-8191 ~ +8191`：
     ```
     raw = (int)lroundf(speed_rpm * (8191.0f / 1000.0f))
     ```
  2. `raw` 即为所需的有符号值，直接作为 14 位二进制补码存入低 14 位。

**代码实现**：

```c
uint16_t data = (16384 + (int)lroundf(speed_rpm * 8.191f)) % 16384 | !enable << 14;
```

### CRC16 校验算法

校验码采用自定义的 CRC16-CCITT 变种，计算过程如下：

1. 初始值 `0x496C`
2. 多项式 `0x1021` (CCITT 标准多项式)
3. **数据预处理**：每个字节在参与计算前需要进行**位反转**（`reverse8`）
4. **移位计算**：左移方式，每次处理 8 位数据
5. **结果后处理**：计算得到的 16 位结果再次进行**位反转**（`reverse16`）

对应代码实现：

```c
uint16_t generate_crc16(const uint8_t *data, int len)
{
    int i, j;
    uint16_t crc = 0x496C;
    const uint16_t poly = 0x1021;
    for (i = 0; i < len; i++) {
        uint8_t byte = reverse8(data[i]);  /* 位反转 */
        crc ^= (uint16_t)byte << 8;
        for (j = 0; j < 8; j++) {
            if (crc & 0x8000)
                crc = (crc << 1) ^ poly;
            else
                crc = crc << 1;
        }
    }
    return reverse16(crc);  /* 结果位反转 */
}
```

其中 `reverse8` 和 `reverse16` 为标准位反转函数。计算得到的 CRC16 值最后以 **小端字节序** 写入帧的第 20、21 字节。

## 项目结构

- `m3508i.h` / `m3508i.c`：核心协议生成代码，提供 `m3508i_build_frame` 函数构造完整指令帧。
- `rs485-converter/`：用于驱动电机的 RS485 转接板 PCB 工程。
- `m3508i-test/`：基于 STM32 的示例项目，展示如何发送指令控制电机。

## 使用示例

```c
#include "m3508i.h"

/* 准备 4 个电机的指令 */
struct m3508i_cmd cmds[4] = {
    { .enable = true, .speed_rpm = 100.0f },   /* ID0 正转 100 rpm */
    { .enable = true, .speed_rpm = -50.0f },   /* ID1 反转 50 rpm */
    { .enable = false },                       /* ID2 禁用 */
    { .enable = true, .speed_rpm = 200.0f }    /* ID3 正转 200 rpm */
};

uint8_t frame[22];
m3508i_build_frame(&frame, 0, &cmds);  /* 构建帧，指定 ID0 为应答电机 */

/* 通过 RS485 发送 frame 数组（22 字节） */
```

发送后，四个电机立即更新转速，ID0 电机会回复一个应答数据包。
